# My Contributions - Heat Production Optimizer

This repository contains a second-semester Software Engineering team project at the University of Southern Denmark. The project is a C# desktop application for heat production optimization based on a Danfoss-style district-heating case.

## Role

C# Desktop Application Developer

I implemented major backend/application logic modules and helped turn the project from a UI concept into a working optimization system with parsed source data, production assets, calculated results, visualized scenarios, and unit tests.

## Main Contributions

- Implemented the CSV parser and source data handling for heat demand and electricity price data.
- Designed and worked with domain models for time-series records, heat demand, electricity prices, production units, and optimization results.
- Implemented major service modules, including Source Data Manager, Asset Manager, Result Data Manager, and Optimizer.
- Built optimization logic for production schedules, expenses, emissions, electricity production, electricity consumption, and profit scenarios.
- Fixed chart/data issues and helped connect service outputs to the Avalonia UI.
- Added xUnit tests for data loading, asset management, result handling, and optimization calculations.
- Debugged core application behavior and helped deliver a technically strong modular desktop application.

## Engineering Areas Demonstrated

- C#/.NET application development
- Domain modeling
- CSV parsing and data processing
- Optimization logic
- Modular service design
- MVVM desktop application structure
- Unit testing with xUnit
- Chart/data integration
